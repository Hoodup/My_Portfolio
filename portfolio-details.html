Python Automated Flight Price Scraping and SMS Notification System

This Python-based application was developed to automate the process of monitoring flight prices for selected cities across the globe. The system performs the following tasks:

  • Automation: Runs a Python script daily to check for flight price updates.
  • Excel Integration: Reads an Excel spreadsheet containing a list of cities that are of interest for the price check.
  • API Scraping: Uses multiple APIs to scrape data for the selected cities, focusing on flight prices and availability across different dates.
  • Price Range Filtering: Filters the flight prices based on a predefined price range and date range for each city.
  • SMS Notifications: When a valid price is found, the system sends an SMS notification to the user with the relevant details, including the flight price and dates.
  • Multiple API Integrations: The application uses several APIs for city data, flight management, and SMS notifications to ensure reliable results.

This project combines automation, web scraping, and API integrations to create a fully functional and hands-off flight price tracking system.

Questions I Asked Myself During the Project

  • How can I automate the process of checking flight prices daily?
  • What APIs are reliable for flight data and SMS notifications?
  • How do I filter flight prices by a specified price range and date?
  • What tools can I use to scrape data from these APIs efficiently?
  • How can I ensure the script runs automatically every day without manual intervention?

Steps I Took to Achieve the Project

  1. Researched and selected the appropriate APIs for flight data and SMS notifications (e.g., flight management APIs, Twilio for SMS).
  2. Created a Python script that reads an Excel file containing cities of interest.
  3. Developed a web scraper to extract flight prices, dates, and availability for the selected cities.
  4. Filtered the data based on predefined price and date ranges to narrow down the results.
  5. Set up a scheduler to run the Python script automatically every day using cron jobs or Task Scheduler.
  6. Integrated an SMS notification system (e.g., using Twilio) to alert me when a suitable flight price is found.
  7. Tested the system thoroughly to ensure that it was reliable and accurate before deploying it for regular use.

Key Takeaways from the Project

  • Effective use of APIs can save a lot of time and effort when building a web scraper.
  • Automating the process of checking for data (such as flight prices) can be extremely useful and efficient.
  • Data filtering is crucial for ensuring the results meet specific criteria (e.g., price range, travel dates).
  • Integrating multiple technologies (Python, APIs, SMS services) can create a powerful and seamless workflow.
  • Scheduling and automating tasks using tools like cron jobs ensure the project runs smoothly without manual intervention.
